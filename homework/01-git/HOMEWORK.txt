init - создает локальный репозиторий в етой папке
clone - копирует репозиторий на комп
branch - создает ветку 
checkout - переходит в ветку
checkout -b - создает и переходит в ветку
add - кидает файл в очередь на закоммичивание
commit - создает чекпойнт, сохраняет изменения
push/pull - закидывает изменения на удаленный репозиторий/
            хавает с удаленного репозитория соответственно
status - чекает изменения
log - выдает историю коммитов
origin - слово, которым комп по умолчанию обзывает захаванный удаленный репозиторий
